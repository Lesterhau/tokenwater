Context

"Training a single large language model can consume as much water as the lifetime drinking water supply of several hundred people." — Li et al., Making AI Less Thirsty (2023)

Methodology

Water estimate: ~100ml per 1,000 tokens. Based on average cooling and electricity generation water intensity for major US data centers (Li et al. 2023; Luccioni et al. 2023). This is a proxy metric — actual consumption varies by provider, model, and data center location. Token counts sourced from API response objects. See About tab for full citations.

What Is This

TokenWater tracks your token usage across AI platforms and estimates the water consumed by data centers to serve your requests. It does not send data anywhere. Everything lives in your browser's localStorage.

AI infrastructure is invisible infrastructure. The servers running Claude, ChatGPT, Gemini, and Grok consume enormous amounts of water for cooling and electricity generation. Microsoft used 6.4 billion liters in 2022 — up 34% year-over-year. Google: +20% same period. Individual users have no way to track their contribution to that demand. TokenWater is that tool.

The Water Math

The proxy: ~100ml of water consumed per 1,000 tokens processed. This estimate is derived from data center power usage effectiveness (PUE) data and cooling water intensity ratios for major US cloud providers. It is a directional estimate, not an exact measurement. Actual consumption varies by provider, region, season, and model architecture.

  • Li, P. et al. (2023). "Making AI Less Thirsty: Uncovering and Addressing the Secret Water Footprint of AI Models." UC Riverside. arxiv.org/abs/2304.03271
  • Luccioni, A.S. et al. (2023). "Power Hungry Processing: Watts Driving the Cost of AI Deployment?" arxiv.org/abs/2311.16863
  • Microsoft 2022 Environmental Sustainability Report
  • Google 2022 Environmental Report

The Behavior Change Mechanic

Social comparison, not guilt. The same mechanic that drove a 2% national reduction in household energy consumption via Opower's neighbor comparison mailings. "You used 34% fewer tokens than the average user this week" is more effective than any amount of environmental messaging. The comparison is built in.

Why Memory Snippets

You can't write directly to most LLMs' memory systems via the API. TokenWater generates personalized snippets — updated weekly with your actual numbers — that you paste into each platform's manual memory settings. It's a workaround, not a hack. It works.

How to Contribute

This is open source. MIT licensed. Zero dependencies. Single HTML file. You do not need to be a developer to contribute.

  • Write a Phase 2 or Phase 3 platform connector — API response format + token field names. Open a PR or just file an issue with the data.
  • Verify or update the water consumption math for a specific provider. We want better per-provider estimates, not just the global average.
  • Translate the app. Japanese, Spanish, Arabic, Hindi are highest-priority languages.
  • Write a browser extension wrapper that auto-captures token counts without manual entry. This is the highest-leverage expansion path.
  • Build the real-time token counting layer via streaming APIs. Currently manual entry only.
  • Add to METHODOLOGY.md with provider-specific PUE and water intensity data.
  • File a GitHub issue if the water estimate is wrong. Cite your source.
